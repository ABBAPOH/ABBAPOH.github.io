Detailed Description

The QDriveController class provides an interface for drives in operating system. .

QDriveController monitors appearing and disappearing of new drives and system and allows to mount and unmout filesystems. On most operating systems when user inserts flash drive or a cd-rom, it is automatically mounted to filesystem and you can monitor this events using this class.

The driveMounted() signal is emitted when drive is mounted to filesystem - when user inserts cd-rom or adds new network drive.

The driveUnmounted() signal is emitted when drive is unmounted from filesystem.

bool QDriveController::mount ( const QString &  device,

Mounts specified device at path.

This function performs mounting operation with the given device. Device can be a GUID or UNC path in windows, filesystem entry located in /dev in Unix or network path (smb:// or afp://) on Mac.

Note
due to implementation, path parameter is ignored for Unix operating systems and is only available on Windows. If empty path is passed on Windows, QDriveController tries to find not used drive letter and mounts using it as path.
