Member Enumeration Documentation

enum Parts::AbstractDocument::State

This enum describes document state.

Enumerator:
NoState 

Default state, means that editor doesn's perform any operations and ready to work

OpenningState 

Means that editor performs long open operation, e.g. it opens huge file or downloads data that should be opened.

SavingState 

Means that editor performs long save operation.

Reipmlement this function to open data located at url in a document.

This funciton is called from setUrl() method when new url should be opened.

void AbstractDocument::save ( const QUrl &  url)
virtualslot

Reimplement to save currently opened document to an url.

Is empty url is passed then document should save data to current url.

Default implementation updates current url and emits urlChanged() signal if necessary.

Note
You should always call default implemetation, otherwise current url won't be updated.

void AbstractDocument::setProgress ( int  progress)
protectedslot

Sets the progress of current operation.

Progress can be changed only when document is in one of the following states: AbstractDocument::OpenningState or AbstractDocument::SavingState. Attempt to change progress in any other state will lead to warning and progress won't be changed.
